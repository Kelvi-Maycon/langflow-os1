# Plano De Acao

Este plano foca em aproximar o LangFlow das duas referencias sem mexer em conteudo nem em logica funcional.

## Fase 1: auditoria e especificacao

1. Mapear a estrutura atual do app.
2. Arquivar a extracao das duas referencias.
3. Definir PRD visual e design system alvo.
4. Preparar pasta de comparacao para screenshots reais.

## Fase 2: fundacao visual

1. Trocar os tokens globais para a nova paleta:
   - sidebar escura premium;
   - canvas warm light;
   - hero indigo/rose;
   - acentos gold, blue pastel, peach.
2. Reescrever a estrutura visual base:
   - shell;
   - sidebar;
   - main content;
   - topbar;
   - cards;
   - botoes;
   - inputs;
   - progress bars;
   - badges.
3. Padronizar raios, sombras, bordas e motion.

## Fase 3: dashboard premium

1. Reestruturar a dashboard para refletir a narrativa das referencias.
2. Aplicar um grid assimetrico:
   - hero principal;
   - coluna de stats e conquista;
   - missoes;
   - jornada;
   - atividade semanal;
   - colecoes;
   - modulos do ciclo.
3. Melhorar a apresentacao da gamificacao usando os dados que ja existem.
4. Adicionar microinteracoes de hover, pulse e reveal.

## Fase 4: consistencia do restante do produto

1. Reader:
   - transformar em workspace premium.
2. Builder:
   - dar cara de sessao guiada e desafio.
3. Flashcard:
   - reforcar visual de revisao e performance.
4. Settings:
   - reorganizar em paineis de configuracao mais elegantes.

## Fase 5: validacao

1. Rodar lint e build.
2. Capturar screenshots do app.
3. Comparar com a regua de referencia.
4. Ajustar ate ficar visualmente mais proximo.

## Criterios de pronto

- Dashboard claramente inspirada nas duas imagens.
- App inteiro com uma linguagem visual unica.
- Gamificacao mais visivel e mais recompensadora no nivel de UX/UI.
- Nenhum fluxo funcional quebrado.
